Задание на курсовую работу по МДК 11.01

Специальность: <ins> 09.02.07 "Информационные системы и программирование" </ins>

Тема курсовой работы: Разработка базы данных для склада автотоваров

Срок представления работы к защите: <ins> 22 декабря 2025 года. </ins>

Перечень подлежащих разработке вопросов:
1. Анализ предметной области. Постановка задачи.

  1.1. Описание предметной области.
    Склад автозапчастей выполняет функции приемки, хранения и отгрузки товаров. Основные задачи системы: фиксация поступлений от поставщиков, контроль текущих остатков, отслеживание места хранения товаров (адресное хранение), оформление          накладных на отгрузку в розничные точки.
    
  1.2. Перечень входных данных.
    Приходные накладные (инвойсы) от поставщиков, номенклатура автозапчастей (артикулы, бренды), данные о контрагентах, запросы на отгрузку.
    
  1.3. Перечень выходных данных.
    Оборотная ведомость (сколько пришло/ушло), отчет об остатках (инвентаризационная опись), накладные ТОРГ-12 (или аналог), список товаров с критическим остатком.
    
  1.4. Ограничения предметной области.
    Нельзя отгрузить товара больше, чем есть на остатке. Каждый товар должен быть привязан к определенной ячейке хранения.
    
  1.5. Взаимодействие с другими программами.
    Интеграция с системой бухучета (экспорт данных о движении средств) и клиентским приложением для кладовщика (Python).
    
2. Инфологическая (концептуальная) модель базы данных.

  2.1. Выделение информационных объектов.
    Сущности: Товар, Категория, Контрагент (Поставщик/Получатель), Движение (Приход/Расход), Позиция движения.
    
  2.2. Определение атрибутов объектов.
    Для товара: артикул, название, код ячейки склада, остаток. Для движения: дата, тип операции (IN/OUT).
    
  2.3. Определение отношений.
    Контрагент — Движение (1:M). Движение — Товар (M:M).
    
  2.4. Построение концептуальной модели.
    Разработка ER-диаграммы складского учета.
    
3. Логическая структура БД.
Нормализация отношений. Учет специфики складских операций (разделение прихода и расхода).

4. Физическая структура базы данных.
Хранение данных в PostgreSQL.

5. Реализация проекта в среде конкретной СУБД.

  5.1. Создание таблиц.
    Создание таблиц products (с полем location_code), contractors (с полем type), stock_movements (шапка накладной), movement_items.
    
  5.2. Создание запросов и представлений.
    Создание View v_stock_balance для отображения реальных остатков с адресами ячеек.
    Написание триггера (или функции), который автоматически пересчитывает поле stock_quantity в таблице товаров при добавлении записи в movement_items (плюсует при приходе, минусует при расходе).
    
  5.3. Разработка интерфейса.
    Прототип АРМ (Автоматизированного Рабочего Места) Кладовщика на Python.
    
  5.4. Назначение прав доступа.
    Роль storekeeper (кладовщик) — операции с товаром. Роль manager — добавление новых контрагентов и отчетов.
    
  5.5. Создание индексов.
    Индексирование поля part_number для быстрого поиска детали и location_code для инвентаризации по полкам.
    
  5.6. Резервное копирование.
