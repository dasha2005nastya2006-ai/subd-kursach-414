1) ".py" - python код, из которого было скомилировано приложение с расширением .exe с помощью библиотеки "auto-py-to-exe".
2) ".sql" - sql запросы на создание таблиц базы данных. Чуть позже в ней появятся запросы для работы с данными. 
3) авторизация в базу данных под разными должностями производится в специальном окне программы, которое сравнивает данные из таблицы,

4) в самой программе функционал зависит от авторизованной роли. то есть, одна роль чисто физически не сможет воспользоваться функциями роли, права доступа которой стоят выше.
Условно, удаление "заказов" доступно только владельцу базы (роль owner из таблицы users).
5) в самом приложении реализована функция смены пароля для каждой записи из таблицы "users", соответсвенно, при желании "сотрудника" пароль можно поменять.
На данный момент пароль везде admin, это сделано для удабюства тестирования. В таблице присутствуют роль "worker", а так же "worker1", для которой я намеренно указал другой пароль для проверки.
Это означает, что при расширении штата сотрудников, их логины и пароли можно будет настраивать более гибким образом - то есть поставить "имя_фамилия - пароль".
6) подсчет суммы покупки высчитывается так же программой. Я посчитал, что это будет быстрее. В дальнейшем это помогло мне реализовать импорт данных в формат .csv, .xlxs и .txt. Это сделано для удобства работы "бухгалтера".
7) в программе реализована функция резервного копирования базы данных. отсюда следует, что можно выполнить загрузку базы данных из этой самой копии.
